The full analysis is implemented in `fgsm_eval.py`, and all outputs are stored under `ogrenci5/`.

## 2) Methodological Implementation
FGSM perturbation is computed as:
\[
x_{\text{adv}} = x + \varepsilon \cdot \mathrm{sign}\left(\nabla_x \mathcal{L}(x,y)\right)
\]

Since project inputs are normalized, adversarial samples are clipped with per-channel normalized bounds:
\[
x_{\text{adv}} \in \left[\frac{0-\mu_c}{\sigma_c}, \frac{1-\mu_c}{\sigma_c}\right]
\]
This keeps the perturbation space consistent with preprocessing and avoids invalid pixel ranges.

## 7) Technical Interpretation
- FGSM substantially degrades accuracy as epsilon increases, confirming expected adversarial sensitivity.
- Adam provides stronger clean accuracy in the tested setup.
- Clipping can improve robustness at specific epsilon ranges (notably for SGD), but this may trade off clean performance.
- Robustness evaluation should report both clean accuracy and attack-time degradation curves, not only a single scalar metric.
