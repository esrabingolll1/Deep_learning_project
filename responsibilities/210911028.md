## Report Contribution (Reducing Over-Confidence)
Label smoothing prevents the model from assigning 100% confidence to the correct class. Mathematically, it turns one-hot labels (1.0 and 0.0) into a distribution (e.g., 0.90 for the correct class, and the remaining 0.10 distributed across the other 9 classes). This helps reduce test failures caused by overfitting. Data augmentation supports the same goal by introducing variability via spatial changes.

### Short Technical Assessment
- Augmentation + label smoothing tends to stabilize validation curves when used with an LR scheduler.
- CosineAnnealingLR enables faster learning early and finer adjustments later.
- ReduceLROnPlateau lowers the learning rate when validation loss plateaus, enabling late-epoch improvements.
