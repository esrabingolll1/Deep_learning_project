## Why Overfitting Is a Problem
There are two possible outcomes during training:

- **Good case:** The model learns general patterns and performs well on unseen data.
- **Bad case (Overfitting):** The model memorizes training examples (including noise) and fails on new data.

If training accuracy is very high (e.g., 95%) but test accuracy is low (e.g., 60%), this is a clear sign of memorization.  
L1 and L2 are used to reduce this gap.

## Core Concepts

### L1 Regularization (Lasso — Sparsity)
L1 adds the absolute sum of weights to the loss:

```text
Total Loss = Original Loss + λ × Σ|w|
```

Its key property is that it can push many weights **exactly to zero**.  
This produces a sparse model, effectively removing less useful connections and forcing the network to focus on important features.

### L2 Regularization (Ridge — Weight Decay)
L2 adds the squared sum of weights to the loss:

```text
Total Loss = Original Loss + λ × Σw²
```

Unlike L1, L2 generally shrinks weights toward zero without setting many to exact zero.  
This yields smoother, more stable decision boundaries.

In PyTorch, this is applied directly via the optimizer `weight_decay` parameter.

### L1 vs L2 Summary
| Aspect | L1 | L2 |
|---|---|---|
| Effect | Pushes many weights to exactly zero | Keeps weights small |
| Outcome | Sparse model | Smooth, stable model |
| Main use | Feature selection / sparsity | General regularization |
| PyTorch implementation | Manual loss term | `weight_decay` |

### λ (Lambda) — Penalty Strength
λ controls regularization strength:

- **Very small λ:** Penalty is weak, overfitting may remain.
- **Very large λ:** Model is over-constrained and underfits.
- **Balanced λ:** Better train-test tradeoff.

Therefore, testing multiple λ values is essential (e.g., 0.01, 0.001, 1e-4, 1e-3).

### Bias-Variance Tradeoff
| Case | Bias | Variance | Meaning |
|---|---|---|---|
| Underfitting | High | Low | Model is too simple |
| Overfitting | Low | High | Model is too complex / memorizes |
| Ideal | Balanced | Balanced | Learns and generalizes |

L1/L2 typically reduce variance by constraining model complexity.  
If λ is too large, bias increases and performance degrades.

### SGD + Momentum
SGD updates parameters using the current gradient.  
Momentum accumulates past gradients as velocity, reducing oscillation and improving directional consistency.  
However, it is more sensitive to learning rate and may require more tuning.

### Adam Optimizer
Adam uses adaptive learning rates per parameter by tracking first and second gradient moments.  
This generally gives:
- less sensitivity to learning-rate choice
- faster convergence in early epochs
- more stable optimization in practical settings
