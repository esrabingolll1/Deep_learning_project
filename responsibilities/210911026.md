## 2) Report Contribution (Course Criteria)

### Dataset Quality (CIFAR-10)
- CIFAR-10 is a balanced benchmark dataset with 10 classes.
- Its balanced class distribution minimizes bias from class imbalance, making it suitable for fair comparisons.

### Cross-Entropy and the Link to MLE
- `CrossEntropyLoss` is used as the classification loss in `train.py`.
- This choice aligns with the Maximum Likelihood Estimation (MLE) perspective: maximizing the probability of the correct class corresponds to minimizing the negative log-likelihood.

### Why ReLU
- ReLU is used as the activation function in `models.py`.
- In the positive region, ReLU keeps gradients around 1, supporting gradient flow in deep networks and reducing vanishing gradient issues.

### Technical Notes
- Test performance increased as capacity increased (Small < Base < Large).
- In this short / subset setting, the gap values are negative, which can occur due to the combined effects of augmentation + scheduler + early stopping.
- Overall, the capacity effect is quantitatively verified; the bias–variance discussion can be strengthened with full-data / longer-epoch runs in the final report.
